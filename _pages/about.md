## 이 블로그는 무엇을 기록하나요

이 블로그는 개발하면서 자주 다시 확인하게 되는 기준, 설정 포인트, 장애 대응 경험을 정리하는 공간입니다.

단순한 개념 요약보다 실제 업무에서 어떤 문제가 생겼고, 왜 그런 결과가 나왔고, 어떻게 확인하고 정리했는지를 남기는 데 초점을 두고 있습니다.

## 주로 다루는 주제

- Spring, JPA, 트랜잭션, 배치 처리 같은 백엔드 개발 주제
- API 설계와 응답 처리, 예외 흐름 정리
- 데이터베이스, 배포, 설정 과정에서 자주 놓치는 포인트
- 운영 중 만난 오류와 트러블슈팅 기록

## 어떤 독자를 위한 블로그인가요

- 실무에서 바로 확인할 수 있는 기준이 필요한 개발자
- 짧더라도 핵심이 정리된 트러블슈팅 기록을 찾는 개발자
- 같은 문제를 다시 만났을 때 빠르게 출발점을 잡고 싶은 개발자

## 글을 정리하는 방식

가능하면 아래 흐름을 기준으로 글을 정리합니다.

1. 어떤 상황에서 문제가 발생했는지
2. 왜 그런 결과가 나왔는지
3. 실제로 어떻게 해결했는지
4. 다음에 비슷한 문제를 만났을 때 무엇부터 확인하면 되는지

필요한 경우 코드, 설정 예시, 체크 포인트를 함께 남겨서 다시 읽었을 때 바로 적용할 수 있도록 정리합니다.

## 운영 기준

이 블로그의 글은 실제 개발 과정에서 다시 참고할 가치가 있는 내용만 남기는 것을 목표로 합니다.

너무 넓게 설명하기보다 특정 상황에서 도움이 되는 판단 기준과 확인 순서를 분명하게 적으려고 합니다.
