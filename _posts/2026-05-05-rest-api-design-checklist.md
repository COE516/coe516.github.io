## 문제 상황

API를 만들다 보면 URL 이름은 신경 쓰는데, 정작 상태 코드, 응답 구조, 멱등성 같은 중요한 부분은 뒤늦게 맞추는 경우가 많습니다.

## 기본 체크리스트

- 자원 중심 URL인가
- HTTP 메서드 의미가 맞는가
- 상태 코드가 일관적인가
- 응답 구조가 예측 가능한가
- 오류 응답도 통일돼 있는가
- 페이징/정렬 규칙이 일정한가

## 자주 생기는 문제

- 동사형 URL 남발
- 성공/실패 응답 구조 불일치
- 200만 계속 반환
- 페이지 API마다 파라미터 이름이 다름

## 정리

좋은 API는 화려한 구조보다 일관성이 강합니다.

- URL, 메서드, 상태 코드 의미를 맞춘다.
- 성공과 실패 응답 형식을 통일한다.
- 같은 종류의 API는 같은 규칙으로 유지한다.
