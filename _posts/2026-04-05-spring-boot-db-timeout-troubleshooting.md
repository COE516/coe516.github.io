## 문제 상황

간헐적으로 DB 연결 타임아웃이 나면 커넥션 풀 문제로만 보이기 쉽습니다. 하지만 실제로는 네트워크 지연, DB 과부하, 잘못된 타임아웃 값이 함께 얽힌 경우가 많습니다.

## 먼저 나눠서 보기

보통은 아래 세 층으로 나눠 보면 빠릅니다.

- 애플리케이션: 풀 고갈, 긴 쿼리, 누수
- 네트워크: 방화벽, DNS, 지연
- DB: 세션 수 부족, 부하, 잠금 대기

## 확인 포인트

### 1. HikariCP 대기 시간 확인

풀에서 연결을 못 받아 timeout이 나는지 먼저 봅니다.

### 2. 느린 쿼리 확인

연결을 못 받는 이유가 사실은 오래 잡고 있는 쿼리일 수 있습니다.

### 3. DB 서버 상태 확인

CPU, 커넥션 수, lock wait, slow query를 함께 봐야 합니다.

## 설정 예시

```yaml
spring:
  datasource:
    hikari:
      connection-timeout: 3000
```

값을 줄이거나 늘리는 것보다, 왜 기다리는지가 먼저입니다.

## 정리

DB 타임아웃은 설정 한 줄로 끝나는 문제가 아닙니다.

- 풀 고갈인지 본다.
- 긴 쿼리나 누수를 의심한다.
- DB 서버 상태를 같이 본다.
- timeout 값 조정보다 원인 분리가 먼저다.
