## 문제 상황

애플리케이션은 뜨는데 DB 연결 시점에 예외가 나거나, 로컬에서는 되는데 운영에서만 연결 실패가 나는 경우가 있습니다. URL, 계정, 비밀번호가 맞아 보여도 계속 안 붙으면 생각보다 자주 숨어 있는 원인이 JDBC 드라이버 버전 불일치입니다.

이 문제는 DB 접속 오류처럼 보이지만 실제로는 **의존성 호환 문제**인 경우가 많습니다.

## 어떤 식으로 드러날까

대표적으로 이런 형태가 많습니다.

- 드라이버 클래스 로딩 실패
- 인증 방식 불일치
- SSL 관련 연결 오류
- 특정 DB 버전에서만 연결 실패
- 로컬은 되는데 운영에서만 예외 발생

즉 설정이 맞아 보여도 드라이버와 서버 버전 조합이 어긋나면 증상이 꽤 애매하게 나옵니다.

## 왜 생길까

보통 아래 경우가 많습니다.

- Spring Boot BOM과 다른 버전을 직접 강제로 넣음
- 오래된 드라이버를 계속 유지함
- MySQL 5.x 기준 설정을 8.x 서버에 그대로 사용함
- 로컬과 서버의 의존성 해석 결과가 다름

## 먼저 볼 것

### 1. 드라이버 버전을 직접 고정했는가

Boot가 관리하는 버전을 벗어나면 호환성 이슈를 직접 떠안게 되는 경우가 많습니다.

### 2. 실제 DB 서버 버전이 무엇인가

드라이버만 최신이라고 좋은 것이 아니라, 실제 서버와 맞는지도 같이 봐야 합니다.

### 3. 클래스명과 URL 옵션이 구버전 기준은 아닌가

예전 설정을 그대로 들고 오면 여기서도 문제가 납니다.

```properties
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

특히 인증, SSL, timezone 관련 옵션은 DB 버전이 바뀌면서 함께 손봐야 하는 경우가 있습니다.

## 실무에서 자주 놓치는 부분

- 드라이버 버전만 바꾸고 URL 옵션은 그대로 둠
- 로컬 캐시 의존성과 CI/운영 의존성 차이를 안 봄
- Spring Boot가 관리하는 버전을 굳이 벗어남
- 서버 DB 버전 확인 없이 최신 드라이버만 넣음

## 짧은 점검 순서

1. Boot 관리 버전을 쓰는지 본다.
2. 실제 DB 서버 버전을 확인한다.
3. 드라이버 클래스명과 URL 옵션을 같이 본다.
4. 로컬과 운영의 의존성 차이가 없는지 확인한다.

## 정리

- JDBC 드라이버 문제는 겉으로 보면 DB 접속 오류처럼 보이지만 실제로는 의존성 문제인 경우가 많다.
- 드라이버 버전과 DB 서버 버전은 같이 봐야 한다.
- Boot가 관리하는 버전을 먼저 신뢰하는 편이 안전하다.
- 버전만 보지 말고 클래스명과 URL 옵션까지 함께 점검해야 덜 헤맨다.
