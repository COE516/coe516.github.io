## 문제 상황

애플리케이션은 뜨는데 DB 연결 시점에 예외가 나거나, 반대로 로컬에서는 되는데 운영에서만 연결 실패가 나는 경우가 있습니다.

이때 생각보다 자주 나오는 원인이 JDBC 드라이버 버전 불일치입니다.

## 어떤 식으로 드러날까

대표적으로 이런 형태가 많습니다.

- 드라이버 클래스 로딩 실패
- 인증 방식 불일치
- SSL 관련 연결 오류
- 특정 DB 버전에서만 연결 실패

즉, URL이 맞아 보여도 드라이버가 DB 서버 버전과 잘 맞지 않으면 이상한 증상이 나옵니다.

## 왜 생길까

보통 아래 경우입니다.

- Spring Boot BOM과 다른 버전을 직접 강제로 넣음
- 오래된 드라이버를 그대로 유지함
- MySQL 5.x 기준 설정을 8.x 서버에 그대로 사용함
- 로컬과 서버의 의존성 해석 결과가 다름

## 점검 포인트

### 1. 의존성 버전 고정 여부 확인

Boot가 관리하는 버전을 쓰는 쪽이 보통 가장 안전합니다.

### 2. DB 서버 버전 확인

드라이버만 최신이라고 좋은 것이 아니라, 실제 DB 서버와 호환되는지도 같이 봐야 합니다.

### 3. 구버전 클래스명 사용 여부 확인

예전 MySQL 드라이버 클래스명을 그대로 쓰는 경우도 있습니다.

```properties
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

## 해결 방향

- 가능하면 Spring Boot 기본 관리 버전을 우선 사용한다.
- 드라이버를 직접 올렸다면 호환 매트릭스를 함께 본다.
- 드라이버 클래스명과 연결 URL 옵션도 같이 정리한다.

## 정리

JDBC 드라이버 문제는 겉으로 보면 DB 접속 오류처럼 보이지만, 실제로는 의존성 문제인 경우가 많습니다.

핵심은 이렇습니다.

- 드라이버 버전과 DB 버전을 같이 본다.
- Spring Boot가 관리하는 버전을 먼저 신뢰한다.
- 클래스명과 URL 옵션까지 함께 점검한다.
- 로컬만 되거나 운영만 안 되면 의존성 차이부터 의심한다.
