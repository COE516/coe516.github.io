## 문제 상황

운영에서는 민감한 값을 `application.yml`에 직접 넣기보다 환경변수로 주입하는 경우가 많습니다.

## 기본 방식

Spring Boot는 환경변수를 프로퍼티와 자동으로 매핑합니다.

예를 들어 아래 프로퍼티는:

```yaml
app:
  api-key: test
```

환경변수에서는 보통 이런 식으로 대응됩니다.

```bash
APP_API_KEY=real-key
```

## 장점

- 배포 환경별 교체가 쉬움
- 비밀값을 코드 저장소 밖으로 뺄 수 있음
- 컨테이너 환경과 잘 맞음

## 주의할 점

- 이름 규칙을 정확히 맞춰야 함
- profile 설정과 함께 쓰일 때 최종 우선순위를 확인해야 함
- 민감한 값은 로그로 노출되지 않게 조심해야 함

## 정리

환경변수 주입은 운영에서 가장 자주 쓰는 방식 중 하나입니다.

- yml보다 우선 적용될 수 있다.
- 이름 매핑 규칙을 알아야 한다.
- 민감한 값 관리에 특히 유용하다.
