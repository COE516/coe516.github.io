## 문제 상황

응답이 느리면 코드 최적화부터 떠올리기 쉽지만, 실제 병목은 DB 쿼리, 외부 API, 직렬화, 캐시 부재 등 다양합니다.

## 먼저 보는 순서

- 느린 쿼리 여부
- 외부 호출 시간
- 불필요한 N+1
- 큰 응답 본문
- 캐시 가능성

## 실무 팁

속도 개선은 "가장 느린 한 구간"을 찾는 게 먼저입니다. 여러 군데를 조금씩 건드리는 것보다 병목 하나를 줄이는 편이 효과가 큽니다.

## 정리

- 측정 없이 튜닝하지 않는다.
- DB와 외부 호출을 먼저 본다.
- 캐시는 마지막이 아니라 조건이 맞으면 빠른 해법이 될 수 있다.
