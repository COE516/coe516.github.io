## 문제 상황

로그인 체크, 로깅, 요청 추적 같은 공통 로직을 어디에 둘지 고민할 때 필터와 인터셉터가 자주 같이 나옵니다.

## 가장 큰 차이

- 필터: 서블릿 레벨
- 인터셉터: 스프링 MVC 레벨

즉, 필터가 더 앞단에서 동작합니다.

## 필터가 잘 맞는 경우

- 인코딩
- 공통 로깅
- 인증 토큰의 1차 처리
- 요청/응답 래핑

## 인터셉터가 잘 맞는 경우

- 로그인 체크
- 권한 확인
- 컨트롤러 진입 전후 공통 처리
- Handler 정보 활용이 필요한 경우

## 정리

- 서블릿 전역 처리면 필터
- 스프링 MVC 흐름 제어면 인터셉터
- 위치와 책임을 나눠서 쓰는 편이 깔끔하다.
